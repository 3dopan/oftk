# 進捗管理（PDCAサイクル）

## Plan（計画） - 毎週月曜、各Phase開始時
1. 今週実装するタスクのリストアップ
2. 依存関係の確認（blockedBy設定）
3. 優先順位の決定
4. 時間見積もり（ユーザーには伝えない）

## Do（実行） - 毎日
1. TaskListで次のタスク確認
2. TaskUpdateでin_progressに変更
3. Task toolでagent起動
4. Agent完了待ち
5. 成果物レビュー
6. TaskUpdateでcompletedに変更

## Check（評価） - 毎週金曜、各Phase完了時
1. 完了タスク数の確認
2. 品質基準の達成度確認
   - コードが動作するか（cargo run）
   - テストが通るか（cargo test）
   - 設計通りに実装されているか
3. パフォーマンス測定（該当Phaseのみ）
4. ユーザーへの進捗報告

## Act（改善） - 問題発見時、週の終わり
1. 問題点の洗い出し
   - ブロッカー
   - 設計ミス
   - 性能問題
2. 解決策の検討
3. 計画の調整
4. 次週のタスク優先順位見直し
