# コミュニケーションルール

## 日次報告（毎日の終わり）
```markdown
## 本日の進捗（YYYY-MM-DD）

### 完了タスク
- [x] Task X.X.X: 〜を実装
- [x] Task X.X.X: 〜をテスト

### 進行中タスク
- [ ] Task X.X.X: 〜を実装中（80%完了）

### 明日の予定
- [ ] Task X.X.X: 〜を実装予定

### ブロッカー
- なし（またはブロッカーの説明）
```

## 週次報告（毎週金曜）
```markdown
## 今週の進捗（Week X）

### 完了したPhase/機能
- Phase X: 〜完了

### 達成した成果物
- 〜ファイル作成
- 〜機能実装

### 来週の計画
- Phase X: 〜を実装予定

### 課題と対策
- 課題: 〜
- 対策: 〜
```

## Agent完了時の確認項目
1. **成果物の存在確認**: Read tool で対象ファイルを読み込み
2. **動作確認**: Bash tool で cargo run または cargo test
3. **コード品質確認**: 構文エラー、設計通りの実装、エラーハンドリング
4. **依存タスクのブロック解除**: TaskUpdate で blockedBy を更新
